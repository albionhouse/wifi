ETPro category definitions

Bot Command and Control

These rules are autogenerated from several sources of known and confirmed active Botnet and other Command and Control hosts. Bot Command and Control block rules are generated and updated daily by ShadowServer, as well as SpyEye Tracker, Palevo Tracker and Zeus Tracker. Port grouped rules offer higher fidelity with destination port modified in rule.

DNS

These rules prevent attacks and vulnerabilities regarding DNS and for abuse of the service by things such as tunneling.

DShield

These IP based rules are updated daily DShield identified attackers and are very reliable.

Exploit

These rules detect direct exploits including Windows exploits (Veritas etc) but exclude exploits like SQL injection and the like.

Malware

These are malware and spyware related rules where there is no clear criminal intent. The threshold for inclusion in this set is typically some form of tracking that stops short of obvious criminal activity. Originally intended to be just spyware, the line between spyware and outright malicious bad stuff has blurred too much since we originally started this set. These rules are for more than just spyware but rest assured nothing you want running on your device or we want running on the network. There are URL hooks for known update schemes, user-agent strings of known malware and a load of others.

Mobile Malware

These are malware and spyware related rules specific to mobile platforms where there is no clear criminal intent.

Shellcode

These rules are for remote shellcode detection. Remote shellcode is used when an attacker wants to target a vulnerable process running on another machine on a local network or intranet. If successfully executed, the shellcode can provide the attacker access to the target machine across the network. Remote shellcodes normally use standard TCP/IP socket connections to allow the attacker access to the shell on the target machine. If the shellcode can establish this connection, it is called a "reverse shell" or a connect-back shellcode because the shellcode connects back to the attacker's machine.

TOR

 These are IP based rules for the identification of traffic to and from TOR exit nodes.

User Agents

These rules are for user agent identification and detection.

WORM

These rules detect traffic indicative of network-based worm activity.
